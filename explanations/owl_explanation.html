üß© What is an Ontology?

Simple Definition:

An ontology is a formal description of a topic/domain

It includes:

  • Classes: Categories of things (Person, Animal, Car)
  • Properties: Relationships (hasWife, owns, livesIn)
  • Constraints: Rules (a person can have only 2 biological parents)
  • Individuals: Actual examples (John, Mary, Dubai)

üì¶ TBox (Terminology Box)

Like a blueprint or dictionary

"What types of things exist?"

Example: Person, hasWife, hasAge

üìã ABox (Assertion Box)

The Actual Data

Like the filled-in form

"What specific things do we know?"

Example: John is a Man, John hasWife Mary

üîç Property Types Explained

Object Properties vs Data Properties

üîó Object Property

Links one THING to another THING

John ‚Üí hasWife ‚Üí Mary
(Both are "things" with their own identity)
üìù Data Property

Links a THING to a VALUE

John ‚Üí hasAge ‚Üí "45"
(45 is just a number, not a "thing")

Property Characteristics

Functional

Can only have ONE value

hasSSN (one SSN per person)

Inverse Functional

Value points to only ONE thing

hasSSN (one person per SSN)

Symmetric

Works both ways equally

isSiblingOf, isFriendOf

Asymmetric

Only works one way

isParentOf (parents aren't children)

Transitive

Chains together

isAncestorOf, isTallerThan

Reflexive

Applies to itself

knows (I know myself)

What can a Reasoner do?

1. Consistency Checking
"Is there any contradiction in the data?"
Example: Finding if someone is both alive and dead
2. Classification
"Build the complete class hierarchy"
Example: Figuring out all parent-child relationships between classes
3. Realisation
"What class does this individual belong to?"
Example: "Mary" ‚Üí discovered to be "Mother" because she has children
4. Inference
"What new facts can we discover?"
Example: If John hasWife Mary ‚Üí then Mary hasHusband John (inverse!)

Steps to Create an Ontology:

  1. Create Ontology: Set IRI (unique web address), add metadata
  2. Define Classes: Create your categories (Person, Animal...)
  3. Create Hierarchy: Set parent-child relationships (Man subClassOf Person)
  4. Add Object Properties: Define relationships (hasWife, teaches)
  5. Add Data Properties: Define attributes (hasAge, hasName)
  6. Set Characteristics: Make properties symmetric, transitive, etc.
  7. Add Individuals: Create actual instances (John, Mary)
  8. Run Reasoner: Let the computer discover new facts!
